# Dae Full-System Review — July 4, 2026

*Five-subsystem audit conducted after weeks of downtime. Core loop, risk/capital,
integrations, self-healing/cognition, and strategies/governance were each reviewed
independently; findings were cross-verified against the source.*

---

## Why Dae went down (the outage chain)

The repo went silent on **March 25, 2026** — the same day as the last commit,
`5adf40c` ("replace hardcoded local paths with env vars for public sharing").
That commit changed `DEEPSTACK_PATH` from a working hardcoded default to `""`.

1. **Fatal import crash.** `deepstack_integration.py` imports
   `core.risk.kelly_position_sizer` at module level; `kalshi_trader/__init__.py`
   imports it unconditionally. With `DEEPSTACK_PATH` unset, `run_bot.py` dies with
   `ModuleNotFoundError: No module named 'core'` **before logging is configured**.
   Verified empirically. → *Fixed in this PR: internal risk fallbacks
   (`risk_fallback.py`) so the missing optional integration degrades loudly
   instead of killing the process.*
2. **launchd crash-looped invisibly.** `KeepAlive` + `ThrottleInterval 10`
   restarted the crashing process every 10 s, forever. `RunAtLoad=false` meant
   that after any reboot she didn't even try. → *Fixed: `RunAtLoad=true`.*
3. **The launcher still hardcoded the old path.** `bot-launcher.sh` pointed at
   `/Users/eddiebelaval/clawd/projects/kalshi-trading` — the env-var scrub only
   touched Python files. → *Fixed: `BOT_DIR` derived from the script's location.*
4. **No layer could see a dead process.** All three self-healing tiers run
   *inside* the bot process. The external watchdog (`scripts/remediate.py`) reads
   a Supabase health row *written by the bot itself* and never checks staleness —
   a dead bot leaves a frozen "healthy" row. No alert ever fired.
5. **The AI repair layer was independently dead.** `agent.py`/`engineer.py` used
   the invalid model ID `claude-sonnet-4-6-20250514` (correct ID has no date
   suffix) — every DaeAgent/DaeEngineer call 404'd. `remediate.py` invoked the
   Claude CLI with a nonexistent `--yes` flag — every Tier-2 session exited
   non-zero before doing anything. → *Both fixed.*

**Deployment note for the Mac:** update the installed plist to point at the
actual checkout (it's the one remaining absolute path), `launchctl bootstrap` it
again, set `DEEPSTACK_PATH` in `.env` if the DeepStack repo is present, and run
`python scripts/preflight.py` before going live.

## Not fixed here — ranked recommendations

### P0 — before live trading resumes

> **Status update (same day):** all five P0s below were addressed in the
> follow-up PR "P0 batch". Item 1 → mark-to-market `_total_balance`, halted
> re-alerting, and a `reset_hwm` command (Telegram: "reset hwm"). Item 2 →
> dead-man staleness check in `remediate.py` + committed
> `deploy/com.id8labs.deepstack-remediate.plist`. Item 3 → UUID
> `client_order_id` on every order. Item 4 → config-disabled strategies join
> `safety_disabled`. Item 5 → self-repair now proposes a PR and never
> auto-merges or restarts; protected files enforced against the git diff;
> import smoke test before commit.

1. **Mark-to-market accounting** (`main.py:1552`). `_total_balance = cash +
   portfolio_value`, but Kalshi's `portfolio_value` is **max payout**, not
   mark-to-market (documented in `kalshi_client.py:529`). The drawdown HWM
   ratchets on this inflated number, persists to SQLite, and never decays — a
   phantom 20% "drawdown" **permanently halts live trading across restarts**
   with one CRITICAL log line. This is the most likely reason a *running* Dae
   would still look dead. Compute MTM (the dashboard code at `main.py:1563`
   already does), add an HWM reset path that distinguishes withdrawals, and
   re-alert periodically while halted. Same issue defeats `min_balance_floor`.
2. **External dead-man's switch.** All monitoring is in-process. Add a cron/
   launchd watchdog that alerts when the Supabase `deepstack_health_status.updated_at`
   (or the log file mtime — `preflight.py` already has this check) is older than
   N minutes. Commit the remediate launchd plist (it exists nowhere in the repo).
3. **Order idempotency** (`kalshi_client.py:458`, `main.py:3500`). Order POSTs
   are retried on timeout with no `client_order_id` — a timeout after acceptance
   double-fills. The parameter already exists; pass a UUID per intent.
4. **Config-disable authority** (`main.py:2223`). Autonomous governance passes
   ALL loaded strategies to the router, so it can re-enable config-disabled
   strategies — including mean_reversion ("structurally invalid", prior 0.9 in
   mean_reverting regime). Pass only config-enabled strategies or add config
   disables to `safety_disabled`.
5. **Defang self-repair auto-merge** (`self_repair.py:292-464`). It `git add -A`
   (stages *everything* dirty), commits, pushes, **auto-merges its own PR**, and
   restarts — gated only by a non-recursive `ast.parse`. One semantically-broken
   repair = crash loop that Tier 3 cannot fix (repair budget spent, process
   dead). Leave the PR open and alert instead; run `python -c "import
   kalshi_trader.main"` + pytest before committing; stage only changed files.
   Protected-files enforcement greps CLI *stdout prose* and grants unrestricted
   `Bash` — enforce with `git diff --name-only` ∩ protected list and reset
   violations.

### P1 — correctness of the money path

6. **calibration_edge enters at the ask but computes edge at mid** with no
   spread gate (`strategies/calibration_edge.py:216,281`). On a wide book the
   true edge at entry is negative and the mark-to-bid stop triggers immediately
   (pay the spread twice + fees). Compute edge against actual entry price; add a
   max-spread gate (~5c). Also: `max_hours_to_expiry: 1500` in config.yaml guts
   the long-dated filter (62 days vs the strategy's own comments); the forced
   pre-settlement exit at `calibration_edge.py:385` contradicts the
   hold-to-settlement EV math; adopted positions with no journal entry default
   `entry_price` to 50 → phantom TP/SL exits.
7. **Settlement backfill after downtime** (`main.py:1697`). Settlements fetch is
   `limit=200`, unpaginated, in-memory watermark, errors logged at *debug* — after
   weeks down, older settlements are lost forever and journal trades stay open,
   corrupting every stat downstream. Paginate with a persisted watermark; log at
   WARNING.
8. **Capital allocator is decorative** (`main.py:3206-3279`). Its `max_size`
   never reaches `calculate_position_size` — phase/regime/reserve/council output
   changes nothing. Pass it through and `min()` with Kelly size.
9. **Kelly/EV wiring**: negative-edge Kelly is floored to a positive 0.005
   (`main.py:1183`) and the EV gate runs on hardcoded priors, not observed
   stats — the 17%-win-rate ruin scenario in slow motion. Return 0 contracts on
   non-positive blended Kelly. Also fix Kelly-squared semantics (f* passed as a
   *fraction multiplier* to the external sizer) and the paper-trade
   contamination of live stats (`performance_tracker.py:297,395,493` — add
   `AND is_paper = 0`; the column exists for exactly this).
10. **API circuit breaker**: half-open "probes" record fake successes
    (`kalshi_client.py:441` — `async with breaker: pass`), 4xx client errors
    count toward opening it, and no 5xx retry exists despite the docstring.
    Gate with a check method; only count 5xx/transport as failures.
11. **Command queue replay** (`command_processor.py:114-192`). Commands are
    marked done *after* execution and the status PATCH failure is swallowed —
    a stuck `pending` command (incl. `place_trade`/`force_close`) re-executes
    every 3s, and weeks-old commands run on restart. Ack first; skip stale.
    Related: `force_close` sells NO at 99c (never fills) and forgets the
    position regardless of order outcome (`command_processor.py:330`).

### P2 — governance and cognition honesty

12. **Regime detection is still constant** on Kalshi data (the 705/705
    `low_vol_calm` pathology persists — thresholds need ~35% price swings in 5
    minutes). With autonomous routing now acting on it, the router + Council of
    Masters are deterministic noise. Also stock strategies are routed by the
    *prediction-market* regime, disabling crisis_alpha exactly when equities
    crash (`market_governor.py:1282`).
13. **Arena poisoning**: synthetic fitness is upserted with `trade_count=100`
    (95% weight), overwriting live observed fitness; loser-relative
    normalization gives fitness 1.0 to the least-bad loser; promotion can flip
    config `enabled: true` with no structural exclusion list (`arena/fitness.py:96`,
    `arena/promotion.py:140`).
14. **lessons.md duplication root cause**: dedup is exact-string match
    (`consciousness.py:163`) while the Haiku heartbeat re-emits paraphrases with
    live numbers embedded (EV drifting -956.1c → -954.1c across "identical"
    lessons) — duplicates accumulate and the line-budget trim then evicts older
    *distinct* lessons. Normalize (strip numbers) before dedup, or key by topic
    slug; unify the two incompatible writers (`agent.py:477` vs
    `consciousness.py:122`).
15. **stock_momentum v2**: trailing stop can never trigger while in profit
    (compares positive pnl against a negative threshold, no high-water mark);
    time exits read metadata that `_place_trade` never stores; short mode sizes
    inverse-ETF positions with the *underlying's* ATR (v1's $149.52 mistake in
    subtler form) (`strategies/stock_momentum.py:461,481,350`).
16. **Restart adoption**: exchange-synced positions get no strategy/entry price →
    exits run disabled mean_reversion logic against a phantom 50c entry
    (`main.py:2522,2725`). Persist strategy+entry on the position record.
17. **Cost/alerting hygiene**: no Anthropic spend caps or 429/529 backoff
    anywhere (switch to the `anthropic` SDK); Telegram send failures downgrade
    all alerting to local log lines with no fallback channel; heartbeat can
    auto-flip `paper_trade=false` (live promotion) with no human gate
    (`heartbeat.py:139`); voided markets are booked as total losses
    (`journal.py:578`); `config.yaml` `analysis.auto_apply_params: true`
    contradicts its own "requires human review" description.
18. **Dead machinery to delete or wire up**: `circuit_breaker.py` (byte-identical
    duplicate of `api_circuit_breaker.py`, unused), the IBKR breaker (created,
    never used), `StrategyHealth.fitness` (read via getattr, never set — the
    fitness feedback loop is dead code), `run_dry_run.py` (crashes on five
    different API mismatches; use `run_bot.py --multi --dry-run`), and the
    second conflicting plist in `deploy/` (different label, venv, and LIVE vs
    PAPER mode — nondeterministic which one owns the machine).

## What's genuinely good

The 60-second trading loop's exception ladder is sound; every HTTP client sets
timeouts; RSA-PSS signing is correct per Kalshi spec; the cents/dollars v2
migration is handled defensively; the fill-confirmation loop handles
cancel-before-fill; the Supabase-vs-config authority bug from Phase 18 is
genuinely fixed; and calibration_edge — the one strategy that earns — has a
defensible core (EV gate → fill-confirmed limit orders → settlement bridge).
The honest summary: **a solid single-strategy trading core wrapped in an
autonomy superstructure that was more dangerous than helpful.** Simplify before
re-arming it.
